### Fraud Detection  
- **S**: The dataset comes from a Kaggle competition on [fraud detection with debit cards](https://www.kaggle.com/competitions/playground-series-s3e4). It consists of **99.8% normal transactions** and only **0.2% fraudulent ones**. Feature names are hidden. 
- **T**: The task is to predict fraudulent transactions. The evaluation metric is the **area under the ROC curve (ROC-AUC)**. 
- **A**: Actions taken: 
  - Performed **EDA** (Exploratory Data Analysis) and identified differences between normal and fraudulent transaction distributions. 
  - Created a **Data Transformer** to preprocess the data and generate new features based on EDA findings. 
  - Trained and fine-tuned multiple models: **Logistic Regression, SVM (with probability calibration), Decision Tree, Random Forest, and a Stacking model**. 
- **R**: The best **ROC-AUC score** achieved was **0.821**, while the best score in the competition was **0.83**.
